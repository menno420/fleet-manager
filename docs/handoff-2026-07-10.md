## Delivery-channel SOP (hard-won — follow it exactly)

**No cross-session send tools exist in this org** (verbatim walls in
`capabilities.md`). Working channels: the git control-files bus (inbox
ORDERs) and **PR comments on a PR the target session is subscribed to** —
but the PR-comment channel **MISSED TWICE** when the comment was sent
mid/late in the target's run (R20 provenance: PR #8's Task-4 scope comment
merged unread; and the PR #20 Product-Forge correction miss, fixed by
follow-up PR #22). The SOP:

1. Comment **the moment the target's PR opens** — never mid- or late-run.
2. Require an explicit **ack token** in the reply.
3. **Verify the ack before the PR merges.**
4. No ack by close-out → the delivery did not happen → **re-dispatch as a
   fresh inbox ORDER** (R20).
