# Spend-instrument model — capped autonomous purchasing for agents

> **Status:** `ideas`

**Idea:** agents suggest purchases through an in-repo purchase-requests
ledger; real autonomous spend flows only through cap-enforcing instruments
(virtual cards with hard monthly limits, prepaid platform balances) under
written standing grants per category, with receipts committed to the repo.
**Why worth having:** it converts "agents can never spend" into a graduated
trust system with hard technical caps — the same shape as the fleet's merge
authority: instrument-enforced, not promise-enforced.
**Unblocks:** paid APIs, domains, ads and other small venture-lab spend
without an owner click per purchase; start suggest-then-one-click, graduate
categories on trust.
