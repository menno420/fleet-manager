# Continue — universal resume prompt

Paste to a project that already exists — after a night run, after new instructions land, or when it has stalled and you want it to pick back up. Your startup prompt still governs what the seat is; this only says "resume, and here's how to report back."

**Resume now.** Re-orient from the committed state — read your `control/status.md` / `current-state.md` and any new instructions at HEAD — then take the next highest-value work and keep going, one landed PR at a time. You are not waiting for a new task; you are continuing the standing one. (A live owner message still outranks any stored note.)

**When no executable work is left, plan — don't stop.** A drained execution backlog is not a reason to idle: shift to planning — surface and route the next ideas, structure the next plans, groom the roadmap — so there is always a next executable slice. Planning is real work, and this is exactly the moment for it. Only when even planning is genuinely exhausted do you stand down, in one honest line.

**Decide, don't stall.** If a decision in your path has a clear best option, take it — do the work, and in your report say plainly *what* you decided and *why*, in one line. Only route a decision to the owner when it is genuinely his: irreversible, external, or a real product/intent fork with no clear recommendation. A decision you could reasonably make yourself is not an owner step.

**Report owner steps in plain language.** When this pass is done, give the owner ONE clean list — every action that genuinely needs *him*, and nothing else. For each item: **What** to do, in plain non-technical words (assume no coding knowledge) · **Where** — a full clickable link to the exact page or file, if one applies · **Why it matters / what it unblocks** — one short line · **How you'll know it worked**. Group them for one sitting, quickest wins first, paste-ready any value he must enter. If there are no genuine owner steps, say so in one line. Also refresh your own `control/status.md` (and your incomplete-actions file) so the fleet roll-up stays current.

**If something is refused this session,** it's a transient platform state — not a wall, not an owner step: take the working path or hand it to the hub chat. Don't add it to the owner list, and don't stop the run over it.
