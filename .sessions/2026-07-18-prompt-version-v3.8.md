## 💡 Session idea

**A `--check-stamp-currency` guard in `regen_b_files.py` that fails when a paste body changed without a `VERSION`/coordinator-version bump.** This session exists precisely because #311 changed the coordinator paste bodies but the generation stamp stayed `v3.7 · 2026-07-15` — a drift class no current check catches (the existing drift checks enforce shared-text identity, not stamp *currency*). The cheap enforcing fix: have the tool compare each seat's live paste-body hash against the committed `projects/<seat>/coordinator-prompt.md` body below the marker; if the body differs but neither the global `VERSION` token nor that seat's registry-header version advanced since the copy was generated, red the run with "body changed without a version bump — restamp." Guard recipe: extend `main()` in `docs/prompts/v3/tools/regen_b_files.py` with a `--check-stamp-currency` mode reusing `paste_body()` + `registry_expected()`; assert `body != committed_body ⇒ stamp advanced`; wire it into the default `run_checks()` so every regen catches the miss inline. Dedup-checked: distinct from #311's `--check-block-identity` idea (that guards cross-seat sameness of the top block; this guards version-stamp freshness after any body edit).
