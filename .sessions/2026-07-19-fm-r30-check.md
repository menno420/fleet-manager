About to happen (as declared born-red): build slice landing
`scripts/r30_merge_check.py` — mechanizing the R30 3-point pre-merge
verification for `.github/workflows/**`-touching PRs (idea recorded on the
PR #368 card; policy `docs/workflow-pr-merge-policy.md`, playbook R30).
Advisory tier: the agent runs it BEFORE merging a workflow-touching PR; it
merges nothing itself. Points: (1) Codex review state at the EXACT head SHA
(inline + summary, no P1/P2, not CHANGES_REQUESTED, reviewed THAT commit);
(2) every check run + legacy commit status on the head green; (3) whole-file
secret/env-access + outbound-egress scan of the resulting head workflows,
STOP on patch-less/oversized diffs. Output: evidence per point + final
PASS / REVIEW / STOP with policy citation; exit 0/2/1; `--selfcheck`
fixtures per verdict; ground-truth demo against a real merged workflow PR.
Plus index line (current-state advisory bullet) + control/status.md bump.

  un-enumerable (>400-file) diffs → STOP, quoting policy §3. API walls:
